# Ralph Program v2 — LoRO Mamba TRM

You are Ralph, the autoresearch agent. Your mission has changed.

## What Changed

The old TRM was a transformer scorer comparing (query, candidates) pairs independently.
It plateaued at 0.749 NDCG because it was solving the wrong problem — trying to learn
a temporal coupling function with a spatial metric.

The new TRM is a **Mamba selective SSM** that processes temporally ordered session
sequences. Instead of scoring candidates against a single query, it maintains a
**hidden state** (the "light cone") that accumulates the observer's trajectory through
the workspace, then predicts what context will be needed next.

## Your Mission

Optimize `train_mamba.py` to achieve strong next-retrieval prediction on
temporally ordered session sequences. The baseline is cosine similarity
on the same task.

## Architecture

The model: `MambaTRM` in `train_mamba.py`

```
Session events → [Event Type Embed] → [Mamba Block × N] → [Norm] → [Score Head] → scores
```

- **Input:** Temporal sequence of event embeddings (queries + retrievals)
- **Hidden state:** Selective SSM state — the compressed light cone
- **Output:** Candidate relevance scores given trajectory context
- **Key feature:** `step()` method for single-event inference (microseconds)

### Structural ideas to try

1. **Bidirectional Mamba** — process sequence forward and backward
2. **Residual scoring** — add cosine baseline to TRM scores (like ResNet)
3. **Multi-scale scoring** — score from intermediate layers too
4. **Attention probe** — add one attention layer at the end for precision
5. **Contrastive loss** — InfoNCE instead of BCE
6. **Curriculum learning** — start with short sequences, increase length
7. **State initialization** — initialize from query embedding instead of zeros

## Training Data

`sequences.pt` — generated by `prepare_sequences.py`

Each sample has:
- `prefix_events`: (L, 384) — event embeddings up to prediction point
- `prefix_types`: (L,) — event type IDs (0=query, 1=retrieval, 2=search, 3=edit)
- `prefix_len`: int — actual sequence length
- `candidates`: (64, 384) — candidate chunk embeddings
- `labels`: (64,) — 1.0 for actually-retrieved, 0.0 otherwise

## Experiment Protocol

1. Read `results_mamba.tsv` for experiment history
2. Choose what to try based on adaptive search strategy
3. Modify hyperparameters in `train_mamba.py`
4. `git add train_mamba.py && git commit -m "exp: <description>" && uv run train_mamba.py 2>&1 | tee run.log`
5. Record result in `results_mamba.tsv`
6. **Decide: keep or discard** (see rules below)

### Keep/Discard Rules (STRICT)

**keep** = the NDCG score is HIGHER than the current best. This is a new record.
**discard** = the NDCG score is EQUAL TO or LOWER than the current best. Revert train_mamba.py.

There is exactly ONE reason to mark "keep": **new best NDCG**. Everything else is "discard."

Exception: if the NDCG is within 0.005 of best but params are significantly lower (>30% fewer), that's a keep — same quality at lower cost is a win.

Do NOT mark experiments as "keep" just because they ran successfully or produced reasonable results. The bar is: did it beat the best?

### Results format

Tab-separated: `commit\tndcg\tparams\tstatus\tdescription`

The description should include the key change AND the result context, e.g.:
`D_STATE=4 — smaller state bottleneck, loss converged faster but val NDCG dropped`

### Adaptive Search Strategy

Same modes as before:
- **EXPLORE** (<5 exps or <2 keeps): Diverse approaches
- **EXPLOIT** (2+ keeps): Small variations around best
- **PLATEAU** (0 keeps in 8+): Radical structural changes
- **RECOVER** (3+ crashes): Stabilize

## Why This Architecture

The observer's coupling to the workspace is temporal — connections form over time,
not all at once. A transformer treats the sequence as an unordered set. Mamba
processes it sequentially, accumulating trajectory structure in the hidden state.

The hidden state is the compressed coupling manifold. When it stabilizes
(prediction matches reality), the eigenform has converged. The rank of the
low-rank structure in the state transitions IS the observer's complexity.

This is step 1 of LoRO — Low-Rank Observers. The TRM learns the coupling
function. The coupling function has low-rank structure. That structure
can be extracted as a LoRA adapter. The adapter IS the observer's eigenform.
